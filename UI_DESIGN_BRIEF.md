# 个人事务 Agent — UI/UX 设计简报

> 用途：喂给设计 AI（Claude design 等），为这个 App 做整体 UI/UX 设计。
> 现状：功能已跑通，但 UI 是 Flutter 默认脚手架，从未做过设计。

## 1. 产品一句话
一个**个人 AI 助理 App**：以**对话（文字 + 语音）**为核心入口，帮我打理**日常生活事务**（待办 / 提醒 / 笔记）和**投资 / 研究**（联网搜索、财经 / 公司 / 风险 / 学术数据查询）。一句话体验：“跟一个懂我的助理说话，它帮我记、帮我查、到点提醒我。”

## 2. 目标用户 & 使用场景
- **用户**：就我自己（个人自用，非多用户产品）。技术背景，但希望 App 本身省心、快、顺手。
- **场景**：手机为主，常**单手、碎片时间、用语音**（走路 / 通勤时按住说“提醒我明天三点开会”）；也会坐下来认真查投研数据。
- **两种气质要并存**：**生活**（轻松、快速、随手）+ **投研**（严肃、信息密度高、要可信）。

## 3. 平台 & 技术约束
- Flutter 构建；**iOS 先行**，**鸿蒙（HarmonyOS）**为第二目标 → 设计尽量**跨平台中性**，别过度依赖纯 iOS 隐喻。
- 当前是 **Material 3** 基础脚手架（默认 AppBar / 底部导航 / 气泡 / ListTile），功能齐但**完全没做设计**。
- **语音是一等输入**（按住说话 PTT）；agent 回复是**逐字流式**出现。

## 4. 现有信息架构（现状）
- **登录页**：邮箱 + 密码，登录 / 注册。
- **主界面**：底部 4 个 tab
  1. **对话**：与 agent 聊天。文字输入框 + 发送键 + **按住说话麦克风**；消息气泡（用户右 / 助手左），助手回复流式；右上角有**模型档位选择**（自动 / 快 / 标准 / 强）和退出。
  2. **待办**：列表。每条 = 完成（打勾）/ 标题 / 截止时间 / 删除。
  3. **提醒**：列表。每条 = 内容 / 触发时间 / 状态 / 完成 / 删除。**到点推送到手机（本地 + 远程，App 关着也收得到）**。
  4. **笔记**：列表。每条 = 标题 / 内容摘要 / 删除。
- 三个列表都支持下拉刷新、空状态。

## 5. 必须在 UI 里安顿好的能力清单
- 对话：文字输入、**语音按住说话**、流式回复、模型档位切换。
- Agent 能调的工具（用户不直接点，但**结果要在 UI 体现**）：
  - 增删改查 **待办 / 提醒 / 笔记**
  - **长期记忆**：它会记住我的偏好 / 事实，并在对话中回忆
  - **联网搜索**：时效信息，带来源
  - **专业数据**：学术论文、公司工商信息、企业风险 / 诉讼、股票 / 基金 / 财报等
- 三类实体的**手动管理**（列表里直接完成 / 删除）。
- 到点**提醒通知**。

## 6. 关键用户流程
1. **语音建提醒（最高频）**：打开 → 按住麦克风 → “提醒我两小时后收衣服” → 松手 → 助手回“已建提醒，18:30 提醒你收衣服” → 到点手机弹通知。
2. **查投研数据**：“招商银行最新净利润多少” → agent 查专业数据 → 给出数据。
3. **联网求证**：“最近黄金价格怎么样” → 联网搜索 → 带来源的回答。
4. **回顾**：切到“待办 / 提醒 / 笔记”看清单，手动勾完成。

## 7. 重点要解决的设计问题（请围绕这些出方案）

**A. 语音输入的呈现与位置（最痛）**
- 现状：右下角一个小麦克风图标，**挤在输入框旁、又贴屏幕边缘**。语音是主输入，这待遇太弱。
- 额外约束：**别放屏幕边角**——iOS 对贴边触摸有系统手势延迟，且边角也不顺手。
- 求：语音作为一等公民怎么呈现？更大 / 更居中的录音按钮？录音中的视觉 + 触觉反馈？按住说话 vs 点按切换？误触与“没说话”保护？

**B. “对话”与“三个清单”的关系**
- 现状：对话能让 agent 建待办 / 提醒 / 笔记，但它们又各自是独立 tab，容易割裂（“我刚说的提醒去哪了？”）。
- 求：如何让“**对话是万能入口、清单是结果沉淀**”这层关系顺畅？是否需要一个**今日概览 / 首页**？底部 4 tab 是否最佳分法？

**C. 双重气质并存（生活 vs 投研）**
- 一个 App 既要轻松记生活琐事，又要严肃看财经 / 公司 / 风险数据。
- 求：同一套界面如何兼容两种信息密度与语气，而不显得“精分”？

**D. Agent 回复的结构化**
- 现状：回复是**纯文本流**。但 agent 实际会“建了提醒 / 查了股票 / 搜了网 / 想起了你的偏好”。
- 求：回复里是否引入**结构化卡片**？（如“✅ 已建提醒”卡、搜索结果带来源卡、财经数据小表格、“记住了…”提示）如何与纯文本流自然混排？

**E. 首屏与空状态**：打开直接进对话，还是先给一个聚合“今日（待办 + 提醒）”的概览？空清单时引导什么？

**F. 模型档位（自动 / 快 / 标准 / 强）**：个人自用要不要露出？放哪不打扰？

## 8. 约束 / 不改
- 功能集**先不扩**（就上面这些）；先把**现有功能**重新组织 + 视觉化。
- 跨平台中性（iOS + 鸿蒙）。
- **建议基调（可改）**：简洁、克制、现代、可信；浅色为主并支持深色；一个沉稳的强调色（现为靛蓝 indigo，可换）。语音与“今日”要有存在感。

## 9. 交付期望
- 信息架构建议（tab / 页面怎么分，是否加“今日”首页）
- 关键页面线框 + 视觉，至少覆盖：
  1. 首屏 / 对话
  2. **语音录音态**（按住说话的反馈）
  3. 一个清单页（如提醒）
  4. 一条**带结构化卡片**的 agent 回复（如“已建提醒”+ 一条财经数据）
- 语音输入交互方案
- 配色 / 字体 / 组件基调（design tokens 更好）
